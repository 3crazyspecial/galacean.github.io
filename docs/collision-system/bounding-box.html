包围类

包围类

在3D世界中，模型一般不会以顶点为级别来计算包围的边界，这样太消耗性能。引擎提供了 AABB、OBB 包围盒以及包围球，来进行边界的模拟，并提供了最小最大等属性值和视椎体交互状态。

类型

1. AABB

AABB 包围盒 （Axis Aligned Bound Box，轴对齐的包围盒）。顾名思义，AABB 类型的包围盒是跟世界轴 XYZ 对齐的，无论模型怎么变换，他都会遍历顶点计算出当前世界坐标系的最小值和最大值，从而求出 AABB。

2. OBB

OBB 包围盒（Oriented Bounding Box，方向包围盒）。与 AABB 不同，OBB 只需要在初始化的时候确定好本地坐标系的最小最大值，之后的计算只需更新矩阵，即可获得最新的方向包围信息。

3. Sphere

包围球，顾名思义，模型的包围类是一个球，也可以通过更新矩阵来得到。适用于那些需要球型碰撞体的场景，如模型本身就接近球体。

方法

生成包围类

先取得模型的最小最大值，引擎的顶点信息都存储在 primitive 属性里面，所以在 Primitive 类里面提供了： getMinMax(modelMatrix?: Mat4, littleEndian = true):{min:Vec3,max:Vec3}，用户可以选择 modelMatrix 来切换本地坐标系还是世界坐标系。得到最小最大值后，选择一个包围类进行初始化，如 OBB：

const obb = new OBB(minLocal, maxLocal, modelMatrix);

获取包围类信息

拿 OBB 包围盒举例,可以获取包围盒的最小最大以及边界顶点值：

const { min,max,corners,minWorld,maxWorld,cornersWorld } = obb;

更新包围盒

OBB 与 AABB 相比，只需要更新矩阵就能获得最新的包围信息：

obb.updateByModelMatrix(modelMatrix);

判断模型是否可见

如果要判断模型是否可见，我们可以检测包围盒是否在视锥体内部（包含或者交叉），即：

 const visible:boolean = obb.isInFrustum(frustumPlanes);

具体相交状态

如果想要判断具体的相交状态：分离、交叉、包含，则可以使用：

const intersectInfo:IntersectInfo = obb.intersectsFrustum(frustumPlanes);
/**
/**
 * 相交信息
 * */
export enum IntersectInfo {
  /** 分离 */
  EXCLUDE,
  /** 交叉 */
  INTERSECT,
  /** 包含 */
  INCLUDE
}
*/

视椎体裁剪

引擎默认使用和自动更新 OBB 包围盒，那些在视锥体外面的模型将不会进行渲染。如果您还想知道有哪些物体进行了剔除，那可以读取 Primitive 的 isInFrustum 属性来知晓某个 primitive 是否可见，或者读取 Primitive 的 boundingBox 进行包围信息的详细查询。
